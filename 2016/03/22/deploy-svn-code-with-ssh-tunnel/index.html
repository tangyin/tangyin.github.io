deploy_svn_code_with_ssh_tunnel

2016-03-22

ssh 隧道

打通内网和外网服务器
情况如下
我有一台外网服务器 外网IP 1.1.1.1
我有一台内网服务器 内网IP 192.168.1.100 (可以访问外网）

我的svn服务跑在内网服务器上，现在我想让外网服务器直接更新svn代码
于是 ssh -N -f -R 3690:127.0.0.1:3690 1.1.1.1 在内网服务器上运行这条ssh隧道的命令
前面是内网的IP和svn服务的端口
第一个port是外网服务器上的监听端口
由于内网服务器是可以正常访问外网的，可以理解为交换机有对外IP
所以隧道打通后，外网的1.1.1.1服务器上就能找到内网这台服务器了，然后做了端口的映射转发

于是我就可以在外网1.1.1.1这台服务器上 使用 svn info svn://localhost/xxxx —username 这种命令了
因为我已经把外网1.1.1.1服务器的3690端口映射到了我们的内网svn服务器上
